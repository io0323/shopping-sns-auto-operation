# Shopping SNS Auto Operation — 設計ドキュメント一式

## ファイル構成

| ファイル | 内容 |
|---|---|
| 01_要件定義.md | 精査後の要件定義。冒頭に変更点サマリ |
| 02_基本設計.md | 精査後の基本設計(エージェント構成・ハーネス設計) |
| 03_詳細設計.md | 精査後の詳細設計(DB・API・バッチ・プロンプト・ログ) |
| 04_ClaudeCodeプロンプト.md | Phase 0〜3の実行用プロンプト集 |
| CLAUDE.md | プロジェクトルートに配置するClaude Code用ルールファイル |

## セットアップ手順

1. 新規リポジトリを作成し、`CLAUDE.md` をルートに配置
2. `docs/` ディレクトリを作り、01〜03の設計書をコピー
3. `04_ClaudeCodeプロンプト.md` の Phase 0 から順にClaude Codeで実行
4. `.env` に楽天アプリID・アフィリエイトID・Anthropic APIキー・月額予算を設定

## 名称について

リポジトリ名は将来の横展開(他アフィリエイト・他SNS)を見据えて中立名としているが、
Phase 1〜3の実装対象は楽天API・楽天ROOM・楽天アフィリエイトに特化している。

## 最重要の前提(必読)

- 楽天ROOMへの**投稿は必ず人間が行う**。本システムの出口は「承認済み下書きのコピー支援」まで
- 楽天アフィリエイトの実績はレポートCSVの**手動ダウンロード→インポート**で取り込む
- 上記2点は楽天の規約に由来する恒久的な制約であり、将来も自動化対象にしない
